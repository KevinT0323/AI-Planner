## 第一部分：批次處理技術 (5題)

### 1. MapReduce 編程模型主要包含哪兩個階段？
A. Extract 和 Load
B. Input 和 Output  
C. Map (映射) 和 Reduce (歸約)  
D. Shuffle 和 Sort  
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** MapReduce 的核心思想是將任務分解為 Map 階段並行處理，再由 Reduce 階段匯總結果。

</details>

### 2. 在 MapReduce 中，Map 階段的主要任務是？
A. 寫入磁碟
B. 排序數據  
C. 將輸入數據轉換為鍵值對 (Key-Value Pairs)  
D. 匯總數據  
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** Map 函數接收輸入並產生中間鍵值對。

</details>

### 3. Apache Spark 相較於 Hadoop MapReduce 的主要優勢是？
A. 只支持 Java
B. 代碼更複雜  
C. 硬碟存儲更便宜  
D. 記憶體內運算 (In-Memory Computing)，速度更快  
<details>
<summary>顯示答案</summary>

**正確答案：D**  
**解析：** Spark 將中間結果存在記憶體中，避免了 MapReduce 頻繁讀寫磁碟的開銷。

</details>

### 4. Spark 中的 RDD 代表什麼？
A. Resilient Distributed Dataset (彈性分散式數據集)  
B. Rapid Data Disk
C. Relational Data Database  
D. Random Data Distribution  
<details>
<summary>顯示答案</summary>

**正確答案：A**  
**解析：** RDD 是 Spark 的核心抽象，代表不可變、可分區的數據集合。

</details>

### 5. Spark DataFrame 與 RDD 的主要區別在於？
A. DataFrame 速度更慢  
B. RDD 支持 SQL
C. DataFrame 不能分散式處理  
D. DataFrame 擁有 Schema (結構化信息)，支持 SQL 查詢與優化  
<details>
<summary>顯示答案</summary>

**正確答案：D**  
**解析：** DataFrame 像資料庫表一樣擁有列名和類型，讓 Spark 能進行執行計畫優化。

</details>

## 第二部分：串流處理與架構 (5題)

### 6. Lambda 架構的核心思想是？
A. 完全依賴雲端
B. 只使用批次處理  
C. 同時維護批次層 (Batch Layer) 和速度層 (Speed Layer)  
D. 只使用串流處理  
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** 批次層保證準確全面，速度層提供低延遲，最後在服務層合併。

</details>

### 7. Kappa 架構與 Lambda 架構的主要區別是？
A. Kappa 架構更複雜  
B. Lambda 架構比較新
C. Kappa 架構只處理文本  
D. Kappa 架構移除了批次層，只保留流處理層  
<details>
<summary>顯示答案</summary>

**正確答案：D**  
**解析：** Kappa 架構認為“批次只是流的特例”，統一用一套流處理代碼。

</details>

### 8. Apache Kafka 的主要角色是？
A. 瀏覽器
B. 分散式消息隊列 (Message Queue) / 流平台  
C. 數據庫  
D. Web 服務器  
<details>
<summary>顯示答案</summary>

**正確答案：B**  
**解析：** Kafka 用於構建實時數據管道和流應用。

</details>

### 9. 串流處理 (Stream Processing) 的數據特徵是？
A. 結構化  
B. 歷史數據
C. 無界、連續、動態  
D. 有界、靜態  
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** 數據像水流一樣源源不斷地產生。

</details>

### 10. Apache Flink 的 "Exactly-Once" 語義保證了？
A. 數據處理速度最快
B. 數據最多處理一次  
C. 數據恰好處理一次，不重不漏  
D. 數據至少處理一次  
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** 這是流處理中最高級別的一致性保證。

</details>

## 第三部分：數據處理工具與平台 (5題)

### 11. Pandas 是 Python 中用於什麼的主要庫？
A. 網絡爬蟲  
B. 數據操作與分析 (DataFrame)  
C. 機器學習訓練
D. 3D 繪圖  
<details>
<summary>顯示答案</summary>

**正確答案：B**  
**解析：** Pandas 提供了強大的表格數據處理能力。

</details>

### 12. NumPy 主要提供什麼功能？
A. 資料庫連接
B. 文字處理  
C. 網頁開發  
D. 高性能多維陣列對象與數值計算  
<details>
<summary>顯示答案</summary>

**正確答案：D**  
**解析：** NumPy 是 Python 科學計算的基礎。

</details>

### 13. R 語言最擅長的領域是？
A. 遊戲開發  
B. 統計分析與數據可視化  
C. 網頁後端
D. 系統編程  
<details>
<summary>顯示答案</summary>

**正確答案：B**  
**解析：** R 是專為統計學家設計的語言。

</details>

### 14. Apache Airflow 是一個什麼工具？
A. 工作流調度與管理平台 (Workflow Management Platform)  
B. 搜索引擎  
C. 數據庫  
D. 文本編輯器
<details>
<summary>顯示答案</summary>

**正確答案：A**  
**解析：** Airflow 以代碼 (Python) 定義、調度和監控工作流 (DAGs)。

</details>

### 15. Scala 語言常與哪個大數據框架一起使用？
A. TensorFlow  
B. MySQL  
C. jQuery
D. Apache Spark  
<details>
<summary>顯示答案</summary>

**解析：** Spark 是用 Scala 編寫的，Scala 開發 Spark 應用最原生。

</details>

## 第四部分：綜合應用 (5題)

### 16. 當需要構建一個實時推薦系統時，最可能的技術組合是？
A. Hadoop MapReduce + Hive  
B. Kafka + Flink (或 Spark Streaming) + Redis  
C. Airflow + Pandas
D. MySQL + Excel  
<details>
<summary>顯示答案</summary>

**正確答案：B**  
**解析：** Kafka 接入流，Flink 實時計算特徵，Redis 提供低延遲讀取。

</details>

### 17. 對於每天產生 TB 級日誌的分析，哪種方案最合適？
A. 存入 HDFS/S3，使用 Spark 或 Hive 進行批次分析  
B. 使用 Python 單機處理  
C. 存入 Excel
D. 人工閱讀  
<details>
<summary>顯示答案</summary>

**正確答案：A**  
**解析：** 大數據場景需要分散式存儲和計算。

</details>

### 18. "Shuffle" 操作通常是性能瓶頸，因為它涉及？
A. 內存讀取  
B. CPU 計算  
C. 網絡 I/O 和 磁碟 I/O  
D. 屏幕顯示
<details>
<summary>顯示答案</summary>

**正確答案：C**  
**解析：** 跨節點傳輸大量數據非常消耗資源。

</details>

### 19. 在 ETL 流程中，Airflow 的感測器 (Sensor) 用於？
A. 感測用戶情緒  
B. 等待某個條件滿足 (如文件到達) 再執行後續任務  
C. 掃描病毒
D. 檢測硬體溫度  
<details>
<summary>顯示答案</summary>

**正確答案：B**  
**解析：** 用於處理依賴外部事件的任務。

</details>

### 20. 為什麼列式存儲 (如 Parquet) 適合 OLAP？
A. 格式簡單
B. 支持事務  
C. 寫入速度快  
D. 讀取特定列時不需要掃描整行，且壓縮率高  
<details>
<summary>顯示答案</summary>

**正確答案：D**  
**解析：** OLAP 通常涉及少數列的聚合計算，列式存儲優勢明顯。

</details>
