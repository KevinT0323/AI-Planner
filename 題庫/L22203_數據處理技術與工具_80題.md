## 第一部分：批次處理技術 (25題)

### 1. MapReduce 編程模型主要包含哪兩個階段？
A. Input 和 Output  
B. Map (映射) 和 Reduce (歸約)  
C. Shuffle 和 Sort  
D. Extract 和 Load

**正確答案：B**  
**解析：** MapReduce 的核心思想是將任務分解為 Map 階段並行處理，再由 Reduce 階段匯總結果。

### 2. 在 MapReduce 中，Map 階段的主要任務是？
A. 匯總數據  
B. 將輸入數據轉換為鍵值對 (Key-Value Pairs)  
C. 排序數據  
D. 寫入磁碟

**正確答案：B**  
**解析：** Map 函數接收輸入並產生中間鍵值對。

### 3. Apache Spark 相較於 Hadoop MapReduce 的主要優勢是？
A. 硬碟存儲更便宜  
B. 記憶體內運算 (In-Memory Computing)，速度更快  
C. 代碼更複雜  
D. 只支持 Java

**正確答案：B**  
**解析：** Spark 將中間結果存在記憶體中，避免了 MapReduce 頻繁讀寫磁碟的開銷。

### 4. Spark 中的 RDD 代表什麼？
A. Relational Data Database  
B. Resilient Distributed Dataset (彈性分散式數據集)  
C. Random Data Distribution  
D. Rapid Data Disk

**正確答案：B**  
**解析：** RDD 是 Spark 的核心抽象，代表不可變、可分區的數據集合。

### 5. Spark DataFrame 與 RDD 的主要區別在於？
A. DataFrame 速度更慢  
B. DataFrame 擁有 Schema (結構化信息)，支持 SQL 查詢與優化  
C. DataFrame 不能分散式處理  
D. RDD 支持 SQL

**正確答案：B**  
**解析：** DataFrame 像資料庫表一樣擁有列名和類型，讓 Spark 能進行執行計畫優化。

### 6. Spark SQL 的主要功能是？
A. 加密數據  
B. 使用 SQL 語法查詢結構化數據  
C. 繪圖  
D. 安裝資料庫

**正確答案：B**  
**解析：** Spark SQL 允許開發者使用 SQL 或 DataFrame API 處理結構化數據。

### 7. 在 MapReduce 的 Shuffle 階段發生了什麼？
A. 數據隨機刪除  
B. 數據從 Map 端傳輸到 Reduce 端，並按 Key 排序與分組  
C. 數據格式化  
D. 數據壓縮

**正確答案：B**  
**解析：** Shuffle 是連接 Map 和 Reduce 的橋樑，涉及網路傳輸和排序。

### 8. Apache Flink 在批次處理中的特點是？
A. 只能做串流處理  
B. 將批次處理視為串流處理的特例 (有界流)  
C. 速度比 Spark 慢  
D. 不支持 SQL

**正確答案：B**  
**解析：** Flink 的核心是流處理引擎，批次數據被視為有邊界的流。

### 9. RDD 的 "Resilient" (彈性/容錯) 特性是通過什麼實現的？
A. 備份所有數據到磁帶  
B. 記錄血緣 (Lineage)，出錯時重新計算丟失的分區  
C. 購買昂貴的硬體  
D. 停止計算

**正確答案：B**  
**解析：** RDD 記住了它是如何從其他 RDD 轉換而來的(血緣)，因此可以重建丟失的數據。

### 10. Spark 的 "Lazy Evaluation" (惰性求值) 意味著？
A. 系統反應很慢  
B. Transformation 操作不會立即執行，直到遇到 Action 操作才觸發計算  
C. 數據不進行計算  
D. 程序員很懶

**正確答案：B**  
**解析：** 這允許 Spark 優化整個執行過DAG (有向無環圖)。

### 11. Hadoop 生態系統中，負責存儲的是 HDFS，負責資源管理的是？
A. MapReduce  
B. YARN  
C. Hive  
D. HBase

**正確答案：B**  
**解析：** YARN (Yet Another Resource Negotiator) 管理集群資源。

### 12. 哪種操作屬於 Spark 的 Action？
A. map  
B. filter  
C. count  
D. flatMap

**正確答案：C**  
**解析：** `map`, `filter`, `flatMap` 是 Transformation，`count` 是會觸發計算並返回結果的 Action。

### 13. MapReduce 適合處理什麼類型的任務？
A. 低延遲的即時查詢  
B. 離線、大規模的批次數據處理  
C. 迭代式機器學習  
D. 圖形渲染

**正確答案：B**  
**解析：** MapReduce 啟動慢，適合吞吐量大的離線任務。

### 14. Spark 用於機器學習的庫是？
A. Spark SQL  
B. Spark Streaming  
C. MLlib  
D. GraphX

**正確答案：C**  
**解析：** MLlib 是 Spark 的分散式機器學習庫。

### 15. 在 Spark 中，寬依賴 (Wide Dependency) 通常會導致？
A. Shuffle 操作  
B. 內存溢出  
C. 加速計算  
D. 數據丟失

**正確答案：A**  
**解析：** 寬依賴（如 groupByKey）需要父 RDD 的數據進行洗牌重組，導致 Shuffle。

### 16. Hive 在 Hadoop 生態中的作用是？
A. 實時流處理  
B. 數據倉庫工具，將 SQL 轉換為 MapReduce 任務  
C. 資源調度  
D. NoSQL 資料庫

**正確答案：B**  
**解析：** Hive 讓熟悉 SQL 的人員能操作 Hadoop 數據。

### 17. 批次處理的一個典型缺點是？
A. 處理能力弱  
B. 高延遲，不適合即時決策  
C. 數據準確性低  
D. 維護成本高

**正確答案：B**  
**解析：** 批次通常需要積累一段時間數據才處理，無法即時。

### 18. Spark 應用程序運行的主入口點是？
A. SparkConf  
B. SparkContext (或 SparkSession)  
C. Driver  
D. Executor

**正確答案：B**  
**解析：** SparkContext 是連接 Spark 集群的入口。

### 19. "WordCount" 是 MapReduce 的經典範例，其 Reduce 階段做什麼？
A. 分割單詞  
B. 將相同單詞的計數相加 (Sum)  
C. 按字母排序  
D. 刪除停用詞

**正確答案：B**  
**解析：** 匯總每個 key (單詞) 對應的所有 value (計數 1)。

### 20. Apache Spark 支援的編程語言不包括？
A. Scala  
B. Java  
C. Python  
D. HTML

**正確答案：D**  
**解析：** HTML 是標記語言，非編程語言。

### 21. DAG 在 Spark 中代表？
A. Data Access Gateway  
B. Directed Acyclic Graph (有向無環圖)  
C. Disk Array Group  
D. Distributed Algorithm Group

**正確答案：B**  
**解析：** Spark 用 DAG 來表示任務的依賴關係和執行流程。

### 22. 在 Spark 中，廣播變數 (Broadcast Variable) 的作用是？
A. 將變數複製到每個節點，減少網路傳輸  
B. 收集所有結果  
C. 刪除變數  
D. 加密變數

**正確答案：A**  
**解析：** 用於在所有節點高效共享只讀的大變數。

### 23. MapReduce 的輸入數據通常存儲在？
A. 關聯式資料庫  
B. HDFS  
C. 記憶體  
D. 磁帶

**正確答案：B**  
**解析：** 雖然可讀其他源，但 HDFS 是最標準的輸入源。

### 24. Spark 的 Standalone 模式是指？
A. 單機運行  
B. Spark 自帶的簡易資源管理器模式  
C. 運行在 YARN 上  
D. 運行在 Kubernetes 上

**正確答案：B**  
**解析：** 這是 Spark 內建的集群模式，不依賴外部資源調度器。

### 25. 小文件問題 (Small Files Problem) 對 Hadoop 的影響是？
A. 沒有影響  
B. 增加 NameNode 內存負擔，降低處理效率  
C. 提高效率  
D. 節省空間

**正確答案：B**  
**解析：** 每個文件都要佔用 NameNode 內存，過多小文件會拖垮 NameNode。

## 第二部分：串流處理與架構 (25題)

### 26. Lambda 架構的核心思想是？
A. 只使用批次處理  
B. 同時維護批次層 (Batch Layer) 和速度層 (Speed Layer)  
C. 只使用串流處理  
D. 完全依賴雲端

**正確答案：B**  
**解析：** 批次層保證準確全面，速度層提供低延遲，最後在服務層合併。

### 27. Kappa 架構與 Lambda 架構的主要區別是？
A. Kappa 架構移除了批次層，只保留流處理層  
B. Kappa 架構更複雜  
C. Kappa 架構只處理文本  
D. Lambda 架構比較新

**正確答案：A**  
**解析：** Kappa 架構認為“批次只是流的特例”，統一用一套流處理代碼。

### 28. Apache Kafka 的主要角色是？
A. 數據庫  
B. 分散式消息隊列 (Message Queue) / 流平台  
C. Web 服務器  
D. 瀏覽器

**正確答案：B**  
**解析：** Kafka 用於構建實時數據管道和流應用。

### 29. 串流處理 (Stream Processing) 的數據特徵是？
A. 有界、靜態  
B. 無界、連續、動態  
C. 結構化  
D. 歷史數據

**正確答案：B**  
**解析：** 數據像水流一樣源源不斷地產生。

### 30. Apache Flink 的 "Exactly-Once" 語義保證了？
A. 數據至少處理一次  
B. 數據最多處理一次  
C. 數據恰好處理一次，不重不漏  
D. 數據處理速度最快

**正確答案：C**  
**解析：** 這是流處理中最高級別的一致性保證。

### 31. Apache Spark Streaming (DStream) 的處理模式是？
A. 純流式 (Native Streaming)  
B. 微批次 (Micro-batching)  
C. 離線批次  
D. 手動處理

**正確答案：B**  
**解析：** Spark Streaming 將流數據切成很小的批次進行處理。

### 32. 相比於 Spark Streaming，Flink 和 Storm 被認為是？
A. 真實流處理 (Native Streaming)  
B. 偽流處理  
C. 批次處理  
D. 數據庫

**正確答案：A**  
**解析：** 它們逐條處理事件，而非微批次，延遲更低。

### 33. 事件驅動架構 (EDA) 的核心是？
A. 定時任務  
B. 組件之間通過事件 (Event) 進行觸發和通信  
C. 數據庫輪詢  
D. 人工干預

**正確答案：B**  
**解析：** 系統反應依賴於狀態變化（事件）的發生。

### 34. Kafka 中的 "Topic" 相當於？
A. 數據庫中的表 (Table) 或消息類別  
B. 用戶名  
C. 密碼  
D. IP地址

**正確答案：A**  
**解析：** 消息根據 Topic 進行分類存儲。

### 35. 窗口 (Windowing) 在串流處理中的作用是？
A. 開啟新視窗  
B. 將無限流數據切分為有限塊以便計算 (如最近5分鐘的平均值)  
C. 顯示圖形界面  
D. 關閉連接

**正確答案：B**  
**解析：** 對於聚合操作（如計數、平均），必須定義窗口範圍。

### 36. 哪種窗口類型是不重疊的？
A. 滑動窗口 (Sliding Window)  
B. 滾動窗口 (Tumbling Window)  
C. 會話窗口 (Session Window)  
D. 所有窗口都重疊

**正確答案：B**  
**解析：** 滾動窗口首尾相接，沒有重疊。

### 37. Kafka 的 "Consumer Group" 允許？
A. 多個消費者並行消費同一個 Topic 的不同分區  
B. 數據被刪除  
C. 只能有一個消費者  
D. 加密數據

**正確答案：A**  
**解析：** 這是 Kafka 實現橫向擴展消費能力的機制。

### 38. "Watermark" (浮水印) 在 Flink 中用於解決什麼問題？
A. 版權保護  
B. 亂序事件 (Out-of-order events) 的處理與事件時間 (Event Time) 推進  
C. 內存洩漏  
D. 磁碟已滿

**正確答案：B**  
**解析：** Watermark 是一種衡量事件時間進展的機制，允許處理延遲到達的數據。

### 39. Apache Storm 的特點是？
A. 專注於極低延遲的實時計算  
B. 專注於批次處理  
C. 使用 SQL  
D. 內存佔用大

**正確答案：A**  
**解析：** Storm 是早期的實時計算框架，強調低延遲。

### 40. 為什麼 Lambda 架構被認為維護成本高？
A. 需要維護兩套代碼邏輯 (批次和串流)  
B. 硬體太貴  
C. 沒人會用  
D. 雲端不支援

**正確答案：A**  
**解析：** 必須確保兩套系統邏輯一致，增加了開發運維負擔。

### 41. Spark Structured Streaming 是為了？
A. 取代 DataFrame  
B. 提供基於 DataFrame/Dataset API 的高級流處理，簡化開發  
C. 處理非結構化數據  
D. 降低速度

**正確答案：B**  
**解析：** 它提供了更統一、更高級的 API，並支持 Event Time。

### 42. "Backpressure" (背壓) 機制在流處理中用於？
A. 增加數據生成速度  
B. 當下游處裡不過來時，反饋給上游降低發送速度，防止崩潰  
C. 壓縮數據  
D. 備份數據

**正確答案：B**  
**解析：** 這是流系統穩定性的關鍵特性。

### 43. Kafka 中的 "Broker" 是指？
A. 客戶端  
B. 伺服器節點  
C. 生產者  
D. 消費者

**正確答案：B**  
**解析：** Kafka 集群由一個或多個 Broker 組成。

### 44. 哪種時間語義通常最準確反映業務事實？
A. 處理時間 (Processing Time)  
B. 事件時間 (Event Time)  
C. 攝取時間 (Ingestion Time)  
D. 系統時間

**正確答案：B**  
**解析：** 事件時間是事件實際發生的時間，不受網絡延遲影響。

### 45. CDC (Change Data Capture) 技術通常與哪種架構結合使用？
A. 靜態網頁  
B. 串流處理 (將資料庫變更視為流)  
C. 批次處理  
D. 文件存儲

**正確答案：B**  
**解析：** CDC 捕獲資料庫變更日誌，實時同步數據。

### 46. 在事件驅動架構中，生產者和消費者通常是？
A. 緊密耦合 (Tightly Coupled)  
B. 解耦 (Decoupled)  
C. 同一進程  
D. 互不相干

**正確答案：B**  
**解析：** 通過消息中間件解耦，提高系統靈活性。

### 47. 狀態管理 (State Management) 在流處理中很重要的原因是？
A. 數據是靜態的  
B. 很多操作 (如聚合、關聯) 依賴過去的歷史信息 (狀態)  
C. 為了節省內存  
D. 為了重啟

**正確答案：B**  
**解析：** 沒有狀態就無法計算累計值或跨事件關聯。

### 48. Kafka 的數據持久化是多少時間？
A. 永久  
B. 默認 7 天 (可配置)  
C. 1 分鐘  
D. 斷電即失

**正確答案：B**  
**解析：** Kafka 基於磁碟存儲，可配置保留策略。

### 49. Flink 的 Checkpoint 機制是為了？
A. 提高速度  
B. 容錯恢復 (Fault Tolerance)  
C. 格式化硬碟  
D. 發送郵件

**正確答案：B**  
**解析：** 定期保存狀態快照，以便故障時恢復。

### 50. 下列哪個不是即時處理的應用場景？
A. 金融風控  
B. 實時推薦  
C. 年終財務報表  
D. IOT 設備監控

**正確答案：C**  
**解析：** 年終報表是典型的歷史數據批次處理。

## 第四部分：綜合應用 (10題)

### 71. 當需要構建一個實時推薦系統時，最可能的技術組合是？
A. MySQL + Excel  
B. Kafka + Flink (或 Spark Streaming) + Redis  
C. Hadoop MapReduce + Hive  
D. Airflow + Pandas

**正確答案：B**  
**解析：** Kafka 接入流，Flink 實時計算特徵，Redis 提供低延遲讀取。

### 72. 對於每天產生 TB 級日誌的分析，哪種方案最合適？
A. 使用 Python 單機處理  
B. 存入 HDFS/S3，使用 Spark 或 Hive 進行批次分析  
C. 人工閱讀  
D. 存入 Excel

**正確答案：B**  
**解析：** 大數據場景需要分散式存儲和計算。

### 73. "Shuffle" 操作通常是性能瓶頸，因為它涉及？
A. CPU 計算  
B. 網絡 I/O 和 磁碟 I/O  
C. 內存讀取  
D. 屏幕顯示

**正確答案：B**  
**解析：** 跨節點傳輸大量數據非常消耗資源。

### 74. 在 ETL 流程中，Airflow 的感測器 (Sensor) 用於？
A. 檢測硬體溫度  
B. 等待某個條件滿足 (如文件到達) 再執行後續任務  
C. 感測用戶情緒  
D. 掃描病毒

**正確答案：B**  
**解析：** 用於處理依賴外部事件的任務。

### 75. 為什麼列式存儲 (如 Parquet) 適合 OLAP？
A. 寫入速度快  
B. 讀取特定列時不需要掃描整行，且壓縮率高  
C. 支持事務  
D. 格式簡單

**正確答案：B**  
**解析：** OLAP 通常涉及少數列的聚合計算，列式存儲優勢明顯。

### 76. NoSQL 與 SQL 資料庫在數據處理中的關係通常是？
A. 互相排斥  
B. 互補 (Polyglot Persistence)  
C. 完全一樣  
D. NoSQL 已完全取代 SQL

**正確答案：B**  
**解析：** 根據數據類型和場景選擇合適的存儲，混合使用。

### 77. 數據科學家在 Spark 中常用 PySpark 是因為？
A. Python 運行速度比 Scala 快  
B. 結合了 Spark 的分散式能力與 Python 的豐富生態  
C. Java 太難  
D. 隨機選擇

**正確答案：B**  
**解析：** 雖然 Python 執行稍慢，但其生態豐富且易用性高。

### 78. 雲原生 (Cloud Native) 數據處理是指？
A. 在本地機房運行  
B. 利用雲端的彈性、託管服務 (如 S3, BigQuery, Serverless) 構建應用  
C. 只用虛擬機  
D. 可以在飛機上運行

**正確答案：B**  
**解析：** 充分利用雲計算特性的架構。

### 79. Avro 格式常與 Kafka 一起使用，因為它？
A. 是文本格式  
B. 支持 Schema Evolution (模式演化) 且序列化體積小  
C. 讀取速度最慢  
D. 只能存儲圖片

**正確答案：B**  
**解析：** 適合流數據的序列化，允許數據結構隨時間變化。

### 80. 在選擇數據處理工具時，首要考慮的因素應該是？
A. 哪個工具最新潮

D. 哪個工具Logo最好看

**正確答案：B**  
**解析：** 技術選型應服務於業務目標和現實條件。
