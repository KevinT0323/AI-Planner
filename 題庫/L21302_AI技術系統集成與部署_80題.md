## 第一部分：核心概念與系統集成 (15題)

### 1. 系統集成的定義是？
A. 將AI模型獨立運行  
B. 將AI模型整合到現有業務系統和IT基礎設施中  
C. 刪除舊系統  
D. 只在雲端運行

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 系統集成是將AI模型整合到現有業務系統和IT基礎設施中的過程，確保AI系統能夠與其他系統無縫協作。</span>

### 2. 系統部署的目的是？
A. 訓練模型  
B. 數據清理  
C. 將訓練好的模型部署到生產環境以提供服務  
D. 特徵工程

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 系統部署是將訓練好的AI模型部署到生產環境，使其能夠為實際業務提供服務。</span>

### 3. 微服務架構的優點不包括？
A. 獨立部署  
B. 技術多樣性  
C. 單點故障風險高  
D. 可擴展性

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 微服務架構通常具有容錯性，降低了單點故障的整體影響，而不是風險高。</span>

### 4. 下列哪項不是微服務架構的挑戰？
A. 服務間通信  
B. 數據一致性  
C. 監控複雜度  
D. 無法獨立部署

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** 微服務的一個主要優點是可以獨立部署，這不是挑戰。</span>

### 5. RESTful API 的特點不包括？
A. 使用HTTP方法(GET, POST等)  
B. 有狀態通信  
C. 資源導向設計  
D. 無狀態通信

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** RESTful API 是無狀態通信(Stateless Communication)。</span>

### 6. 適合內部微服務高效通信的協議是？
A. REST  
B. GraphQL  
C. gRPC  
D. FTP

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** gRPC 基於高效的二進制協議，非常適合內部服務之間的通信。</span>

### 7. GraphQL的主要優勢是？
A. 靈活查詢，減少過度獲取  
B. 只能用於數據庫  
C. 傳輸速度最慢  
D. 不支持HTTP

<span style="color: white">**正確答案：A**</span>  
<span style="color: white">**解析：** GraphQL 允許客戶端精確請求所需數據，避免了數據的過度獲取(Over-fetching)。</span>

### 8. ETL流程中的 "T" 代表？
A. Transfer (傳輸)  
B. Transform (轉換)  
C. Test (測試)  
D. Target (目標)

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** ETL 代表 Extract (提取), Transform (轉換), Load (載入)。</span>

### 9. 適用於實時數據流處理的工具是？
A. Hadoop  
B. Apache Spark (Batch模式)  
C. Apache Flink  
D. Excel

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** Apache Flink 和 Kafka, Storm 都是常見的實時數據流處理工具。Hadoop主要用於批次處理。</span>

### 10. 系統介面規範中，常用的結構化數據格式是？
A. JPEG  
B. MP3  
C. JSON  
D. TXT

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** JSON 是最常用的結構化數據交換格式。</span>

### 11. Protocol Buffers 的特點是？
A. 文本格式，可讀性高  
B. 高效二進制格式，體積小  
C. 只能在 Google 使用  
D. 不支持跨語言

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Protocol Buffers 是 Google 開發的高效二進制序列化格式。</span>

### 12. 系統整合時，與資料庫整合的方式不包括？
A. 直接查詢  
B. 數據同步  
C. 共享密碼本  
D. 數據複製

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 常見的資料庫整合方式包括直接查詢、數據同步和數據複製。共享密碼本不是整合方式。</span>

### 13. 下列哪種不是常見的資料庫類型？
A. 關聯式資料庫 (RDBMS)  
B. NoSQL  
C. 時序資料庫  
D. 隨機資料庫

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** 資料庫類型包括關聯式、NoSQL、時序資料庫等，沒有隨機資料庫。</span>

### 14. 異步消息隊列整合常用的工具是？
A. MySQL  
B. Kafka / RabbitMQ  
C. Nginx  
D. Docker

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Kafka 和 RabbitMQ 是常用的消息隊列中間件，用於異步通信。</span>

### 15. 第三方雲端服務整合的考量不包括？
A. API限制  
B. 成本控制  
C. 服務依賴  
D. 員工心情

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** 整合第三方服務需考量API限制、成本、依賴性等技術和商業因素。</span>

## 第二部分：部署環境與策略 (15題)

### 16. 本地部署(On-Premise)的最大優勢通常是？
A. 彈性擴展  
B. 數據控制與安全  
C. 維護簡單  
D. 初始成本低

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 本地部署數據不出本地，適合對數據安全和隱私有嚴格要求的場景。</span>

### 17. 雲端部署(Cloud)的優點是？
A. 初始硬體投資大  
B. 彈性擴展與按需付費  
C. 維護最複雜  
D. 數據必須在本地

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 雲端部署提供彈性資源，按使用量付費，且由雲廠商負責基礎設施維護。</span>

### 18. 混合部署(Hybrid)是指？
A. 同時使用多種程式語言  
B. 結合本地部署和雲端部署  
C. 強制將數據公開  
D. 不使用網路

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 混合部署結合了本地基礎設施和公共雲服務。</span>

### 19. 邊緣部署(Edge)為了降低延遲，將模型部署在哪裡？
A. 中央機房  
B. 雲端數據中心  
C. 設備端 (IoT, 手機等)  
D. 備份伺服器

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 邊緣部署將計算能力推向網絡邊緣，即靠近數據源頭的設備端。</span>

### 20. Docker技術的核心概念是？
A. 虛擬機  
B. 容器化  
C. 無伺服器  
D. 分散式數據庫

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Docker 是一種流行的容器化技術。</span>

### 21. Dockerfile 的作用是？
A. 存放數據  
B. 定義鏡像構建過程  
C. 管理網絡  
D. 監控日誌

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Dockerfile 包含了一系列指令，用於自動構建 Docker 鏡像。</span>

### 22. Kubernetes (K8s) 的主要功能是？
A. 代碼編寫  
B. 容器編排與管理  
C. 數據標註  
D. 圖像處理

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Kubernetes 是用於自動部署、擴展和管理容器化應用程序的開源系統。</span>

### 23. Kubernetes 中的最小部署單元是？
A. Container  
B. Pod  
C. Node  
D. Cluster

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Pod 是 Kubernetes 中可以創建和管理的最小可部署計算單元。</span>

### 24. CI/CD 中的 CI 代表？
A. Continuous Improvement (持續改進)  
B. Continuous Integration (持續整合)  
C. Computer Interface (電腦介面)  
D. Cloud Infrastructure (雲端基礎設施)

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** CI 代表 Continuous Integration (持續整合)。</span>

### 25. 藍綠部署(Blue-Green Deployment)的主要優點是？
A. 節省資源  
B. 零停機切換與快速回滾  
C. 不需要兩套環境  
D. 測試時間長

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 藍綠部署通過維護兩套相同的環境來實現無縫切換，若新版本有問題可立即切換回舊環境。</span>

### 26. 金絲雀部署(Canary Deployment)是指？
A. 用金絲雀測試空氣品質  
B. 一次性全量發布  
C. 逐步將流量切換到新版本  
D. 只在內部環境部署

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 金絲雀部署是先將小部分流量引導至新版本，驗證無誤後再逐步擴大流量。</span>

### 27. 常用的模型服務框架不包括？
A. TensorFlow Serving  
B. TorchServe  
C. Seldon Core  
D. Adobe Photoshop

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** Photoshop 是圖像處理軟體，不是模型服務框架。</span>

### 28. 模型壓縮技術中的 "量化 (Quantization)" 是指？
A. 減少模型層數  
B. 降低數據精度 (如從FP32到INT8)  
C. 刪除神經元  
D. 增加數據量

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 量化是通過降低數值精度（例如使用整數代替浮點數）來減小模型大小和提高推理速度。</span>

### 29. 語義版本控制 (Semantic Versioning) 的格式通常是？
A. v1  
B. 2023.01.01  
C. Major.Minor.Patch (主版本.次版本.修訂版本)  
D. Final.Final_v2

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 語義版本控制規範使用 主版本號.次版本號.修訂號 的格式。</span>

### 30. 模型蒸餾 (Distillation) 的目的是？
A. 增加模型複雜度  
B. 將大模型(Teacher)的知識轉移到小模型(Student)  
C. 增加訓練時間  
D. 加密模型

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 蒸餾是一種模型壓縮技術，通過訓練小模型來模仿大模型的行為。</span>

## 第三部分：監控與維護 (15題)

### 31. 模型性能監控中的 "線上準確率" 通常如何獲取？
A. 直接計算  
B. 通過用戶反饋或事後標註獲取真實標籤後計算  
C. 猜測  
D. 使用訓練集數據

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 線上真實數據通常沒有即時標籤，需要通過用戶行為（如點擊）或延遲獲取的真實結果（如事後標註）來計算準確率。</span>

### 32. 延遲監控中的 "P99延遲" 代表什麼？
A. 平均延遲  
B. 99% 的請求延遲低於該數值  
C. 99% 的請求延遲高於該數值  
D. 最低延遲

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** P99 延遲表示 99% 的請求響應時間都快於這個數值，反映了系統在長尾情況下的性能。</span>

### 33. QPS 指標是指？
A. Quality Per Second (每秒質量)  
B. Queries Per Second (每秒查詢數)  
C. Questions Per System (每系統問題數)  
D. Quiet Period Start (靜默期開始)

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** QPS (Queries Per Second) 是衡量系統吞吐量的重要指標。</span>

### 34. 數據漂移 (Data Drift) 監控主要關注？
A. 硬體老化  
B. 輸入數據分佈隨時間的變化  
C. 代碼版本變化  
D. 網絡帶寬

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 數據漂移是指線上輸入數據的統計分佈與訓練數據相比發生了變化。</span>

### 35. 概念漂移 (Concept Drift) 通常表現為？
A. 數據格式錯誤  
B. 模型性能下降，輸入數據分佈可能未變但業務規律變了  
C. 服務器當機  
D. 日誌丟失

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 概念漂移是指輸入與輸出之間的映射關係發生了變化（例如用戶偏好改變），導致模型預測能力下降。</span>

### 36. 基礎設施監控不包括？
A. CPU使用率  
B. 記憶體泄漏  
C. 用戶滿意度調查  
D. 磁碟IO

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** 用戶滿意度屬於業務指標，不屬於IT基礎設施監控。</span>

### 37. 常用的集中式日誌管理方案是？
A. 記事本  
B. ELK Stack (Elasticsearch, Logstash, Kibana)  
C. Word文檔  
D. 本地文件夾

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** ELK Stack 是業界廣泛使用的日誌收集、存儲和分析解決方案。</span>

### 38. 模型性能下降時，最直接的應對措施通常是？
A. 重新啟動伺服器  
B. 模型重訓 (Retraining)  
C. 修改前端界面  
D. 降低價格

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 當模型因數據漂移或概念漂移導致性能下降時，通常需要使用新數據對模型進行重新訓練。</span>

### 39. 線上學習 (Online Learning) 的特點是？
A. 每年更新一次模型  
B. 持續從新流入的數據中學習並更新模型  
C. 只使用離線數據  
D. 不需要監控

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 線上學習是模型能夠即時地從新的數據流中學習 updates，適應性強。</span>

### 40. 災難性遺忘 (Catastrophic Forgetting) 是指？
A. 忘記備份數據  
B. 模型在學習新知識時忘記了舊知識  
C. 硬碟損壞導致數據遺失  
D. 開發者忘記代碼邏輯

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 這是線上學習或增量學習中的常見問題，模型在擬合新數據時破壞了原有的參數結構。</span>

### 41. A/B 測試的主要目的是？
A. 測試網路連通性  
B. 比較不同模型或策略版本的效果  
C. 檢查語法錯誤  
D. 備份數據

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** A/B 測試將流量分成兩組，分別使用不同版本，以統計學方法比較其效果。</span>

### 42. 日誌分析的作用不包括？
A. 錯誤排查  
B. 性能分析  
C. 自動寫代碼  
D. 用戶行為分析

<span style="color: white">**正確答案：C**</span>

### 43. 服務可用性通常通過什麼來監控？
A. 健康檢查 (Health Check) 接口  
B. 詢問客服

D. 每年一次的審計

### 44. 關於模型重訓的觸發條件，下列何者不正確？
A. 檢測到顯著的數據漂移

### 45. Splunk 主要用於？

<span style="color: white">**解析：** Splunk 是一個強大的機器數據和日誌分析平台。</span>

## 第四部分：最佳實踐與安全 (15題)

### 46. 水平擴展 (Horizontal Scaling) 是指？
A. 升級單台伺服器的配置 (如加RAM)  
B. 增加伺服器實例的數量  
C. 購買更大的顯示器  
D. 延長工作時間

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 水平擴展是通過增加節點數量來提升系統處理能力。</span>

### 47. 垂直擴展 (Vertical Scaling) 是指？
A. 增加伺服器數量  
B. 增強單台伺服器的資源 (如CPU, RAM)  
C. 把伺服器疊起來  
D. 減少功能

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 垂直擴展是提升單個節點的硬體性能。</span>

### 48. 負載均衡 (Load Balancing) 的作用是？
A. 增加系統負載  
B. 將請求流量分發到多個伺服器  
C. 備份數據  
D. 加密數據

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 負載均衡器將流量均勻分配給後端伺服器，避免單點過載，提高系統可用性。</span>

### 49. 在安全性考量中，數據傳輸安全通常依賴？
A. HTTP  
B. HTTPS (TLS/SSL加密)  
C. 明文傳輸  
D. FTP

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 使用 HTTPS 可以加密傳輸中的數據，防止竊聽和篡改。</span>

### 50. API 安全措施不包括？
A. 身份認證 (Authentication)  
B. 授權控制 (Authorization)  
C. 速率限制 (Rate Limiting)  
D. 公開所有接口文檔給所有人

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** 雖然公開文檔是好的，但API本身需要認證、授權和限流來保護。敏感接口不應隨意公開。</span>

### 51. 對抗攻擊 (Adversarial Attack) 是指？
A. 黑客物理破壞伺服器  
B. 通過精心設計的輸入誤導AI模型做出錯誤判斷  
C. 競爭對手降價  
D. 断网

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 對抗攻擊是針對機器學習模型的攻擊方式，通過微小的擾動干擾模型輸出。</span>

### 52. 災難恢復計畫 (DRP) 的核心要素不包括？
A. 備份策略  
B. 恢復流程  
C. 定期演練  
D. 祈禱

<span style="color: white">**正確答案：D**</span>  
<span style="color: white">**解析：** 災難恢復依靠備份、流程和演練，而不是運氣。</span>

### 53. 模型推理優化中的 "批次處理 (Batching)" 可以？
A. 降低單個請求的延遲  
B. 提高系統整體的吞吐量 (Throughput)  
C. 減少模型大小  
D. 增加準確率

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 批次處理可以利用硬體（如GPU）的並行計算能力，顯著提高吞吐量，但可能會輕微增加單個請求的延遲。</span>

### 54. 知識庫在運維中的作用是？
A. 存儲日誌  
B. 記錄最佳實踐和經驗，方便故障排除和知識傳承  
C. 備份代碼  
D. 管理聊天記錄

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 知識庫用於沉澱團隊經驗，幫助快速解決已知問題。</span>

### 55. TensorRT 是哪家公司推出的推理優化引擎？
A. Google  
B. NVIDIA  
C. Microsoft  
D. Intel

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** TensorRT 是 NVIDIA 針對自家 GPU 推出的深度學習推理優化庫。</span>

### 56. ONNX Runtime 的主要優勢是？
A. 只能在 Windows 運行  
B. 跨平台、跨框架的高性能推理  
C. 只能運行 PyTorch 模型  
D. 開發介面最複雜

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** ONNX Runtime 支持多種硬體加速器和多種框架導出的 ONNX 模型，提供統一的推理體驗。</span>

### 57. 速率限制 (Rate Limiting) 的目的是？
A. 限制網速  
B. 防止API被濫用或DoS攻擊，保護後端服務  
C. 限制員工上網  
D. 降低準確率

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 限流可以防止某個用戶或IP發送過多請求壓垮服務。</span>

### 58. 身份認證 (Authentication) 確認的是？
A. 你能做什麼 (權限)  
B. 你是誰 (身份)  
C. 你在哪裡  
D. 你的設備

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 認證是確認用戶身份的過程（你是誰），授權是確認權限（你能做什麼）。</span>

### 59. 授權控制 (Authorization) 確認的是？
A. 你是誰  
B. 你是否有權限執行特定操作  
C. 你的密碼是否正確  
D. 你的註冊時間

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 授權是在認證之後，決定用戶是否有權訪問特定資源。</span>

### 60. DevOps 與 MLOps 的主要區別在於？
A. MLOps 不需要寫代碼  
B. MLOps 額外關注模型、數據的版本管理與持續訓練  
C. DevOps 只關注硬體  
D. 沒有區別

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** MLOps 將 DevOps 的原則應用於機器學習，並增加了對數據、模型生命週期的管理。</span>

## 第五部分：綜合應用 (20題)

### 61. 當需要全球服務且快速擴展時，最佳部署選擇是？
A. 本地部署  
B. 雲端部署  
C. 離線部署  
D. 紙本作業

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 雲端部署具有全球節點和彈性擴展能力，適合此場景。</span>

### 62. 對於醫療數據等高度敏感且有法規限制的場景，首選部署是？
A. 公有雲  
B. 本地部署 (On-Premise)  
C. 邊緣部署  
D. 混合雲

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 為了嚴格控制數據隱私和符合法規，本地部署通常是首選。</span>

### 63. 自動駕駛汽車的模型部署屬於哪種場景？
A. 雲端部署  
B. 邊緣部署 (Edge)  
C. 本地部署  
D. 混合部署

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 自動駕駛需要極低延遲和離線能力，必須在車載設備端（邊緣）運行。</span>

### 64. 如果希望新版本上線風險最低，且資源充足，應選擇？
A. 直接覆蓋更新  
B. 藍綠部署  
C. 停機更新  
D. 手動更新

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 藍綠部署可以瞬間切換且依賴於驗證過的環境，風險極低，但需要雙倍資源。</span>

### 65. 如果資源有限，但希望逐步驗證新版本，應選擇？
A. 藍綠部署  
B. 金絲雀部署  
C. 重裝系統  
D. 暫停服務

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 金絲雀部署不需要雙倍資源，且能控制風險，適合逐步驗證。</span>

### 66. Kafka 在 AI 系統架構中通常扮演什麼角色？
A. 數據庫  
B. 計算引擎  
C. 消息緩衝與解耦  
D. 前端展示

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** Kafka 作為消息隊列，用於緩衝高吞吐量的數據流，並解耦生產者和消費者。</span>

### 67. 在微服務中，如果一個服務掛了，為了防止影響整個系統，通常會設計？
A. 熔斷機制 (Circuit Breaker)  
B. 無限重試  
C. 報警  
D. 自動關機

<span style="color: white">**正確答案：A**</span>  
<span style="color: white">**解析：** 熔斷機制可以防止故障擴散（雪崩效應）。</span>

### 68. 模型服務化 (Model Serving) 的核心目標是？
A. 訓練模型  
B. 將模型封裝為標準服務接口 (如API) 供調用  
C. 壓縮模型  
D. 加密模型

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 模型服務化是將模型變為可用的軟體服務。</span>

### 69. 為什麼需要對輸入數據進行驗證？
A. 為了收集數據  
B. 防止異常數據導致模型崩潰或輸出錯誤  
C. 為了增加延遲  
D. 美化數據

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 輸入驗證是防禦性編程的一部分，確保模型接收到的數據符合預期格式和範圍。</span>

### 70. 4xx 錯誤碼通常表示？
A. 伺服器內部錯誤  
B. 客戶端請求錯誤  
C. 成功  
D. 重定向

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** HTTP 4xx 代表客戶端錯誤（如 400 Bad Request, 404 Not Found）。</span>

### 71. 5xx 錯誤碼通常表示？
A. 客戶端錯誤  
B. 伺服器端錯誤  
C. 成功  
D. 網絡錯誤

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** HTTP 5xx 代表伺服器端錯誤（如 500 Internal Server Error）。</span>

### 72. 哪種資料庫適合存儲大量的物聯網傳感器數據？
A. 關聯式資料庫  
B. 時序資料庫 (Time-Series DB)  
C. 圖資料庫  
D. 內存資料庫

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 時序資料庫專為帶時間戳的序列數據優化，適合IoT場景。</span>

### 73. Redis 在 AI 系統中常用於？
A. 永久存儲大文件  
B. 高速緩存 (Cache) 與特徵存儲  
C. 訓練模型  
D. 關係查詢

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Redis 是高性能的內存鍵值存儲，常用於緩存熱點數據或作為 Feature Store 的低延遲層。</span>

### 74. Git 版本控制主要用於管理？
A. 大規模二進制數據  
B. 源代碼和配置文件  
C. 編譯後的執行檔  
D. 硬件資產

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Git 最適合管理文本與代碼。</span>

### 75. 什麼情況下需要回滾 (Rollback)？
A. 新版本性能優異  
B. 新版本上線後發現嚴重Bug或性能指標下降  
C. 到此一遊  
D. 定期操作

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 回滾是發布失敗時的止損措施。</span>

### 76. 預處理服務 (Preprocessing Service) 的作用是？
A. 存儲結果  
B. 將原始數據轉換為模型需要的特徵格式  
C. 顯示結果  
D. 訓練模型

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 預處理服務負責特徵提取、歸一化等操作，是連接業務數據和模型輸入的橋樑。</span>

### 77. 後處理服務 (Post-processing Service) 的作用是？
A. 數據收集  
B. 將模型輸出的概率或數值轉換為業務可理解的結果  
C. 預處理  
D. 日誌記錄

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** 後處理將模型的原始輸出（如logits）轉換為標籤、置信度過濾或業務指令。</span>

### 78. IaaS (Infrastructure as a Service) 提供的主要是？
A. 應用軟體  
B. 開發平台  
C. 基礎計算資源 (虛擬機, 存儲, 網絡)  
D. 代碼託管

<span style="color: white">**正確答案：C**</span>  
<span style="color: white">**解析：** IaaS 提供最底層的基礎設施資源，如 AWS EC2。</span>

### 79. SaaS (Software as a Service) 的例子是？
A. Google Colab  
B. ChatGPT 網頁版 / Google Docs  
C. Linux Kernel  
D. Docker Engine

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** SaaS 讓用戶直接使用軟體應用，無需關心底層維護。</span>

### 80. 在容器化環境中，Pod 的生命週期通常是？
A. 永久的  
B. 短暫的 (Ephemeral)，隨時可能被銷毀重建  
C. 與硬體綁定  
D. 不可停止的

<span style="color: white">**正確答案：B**</span>  
<span style="color: white">**解析：** Cloud Native 設計理念中，實例是可替換的，隨時可能因擴縮容或故障恢復而重建。</span>
